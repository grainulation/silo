Reusable knowledge for research sprints. Grab a starter pack in one command.

$ npx @grainulator/silo pull compliance --into ./claims.json

What Silo Does

Shared Claim Libraries

Reusable constraint sets that encode hard-won knowledge. HIPAA rules, migration risks, architecture trade-offs.

Sprint Templates

Pre-built research questions with seeded claims. Start a migration sprint with known risks already loaded.

Import / Export

Pull claims from any pack into your sprint. Push your findings back for the next team to use.

Knowledge Packs

Version-controlled bundles. Publish packs from your best sprints. Install packs from your team.

Quick Start

# See what's available
silo packs

# Pull compliance constraints into your sprint
silo pull compliance --into ./claims.json

# Search across all stored knowledge
silo search "encryption at rest"

# Store your findings for reuse
silo store "q4-findings" --from ./claims.json

Design Principles

Zero Dependencies

Node.js built-ins only. No database, no network calls, no accounts. Works offline, works forever.

Filesystem Storage

Everything is JSON files in ~/.silo. Inspect, back up, version control, or delete with standard tools.
