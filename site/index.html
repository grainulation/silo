How to use silo
1

List available packs

See what knowledge packs are available -- compliance, migration, architecture, and more.

$ npx @grainulator/silo packs
2

Pull into your sprint

Import a pack's claims directly into your claims.json. They merge cleanly with existing claims.

$ npx @grainulator/silo pull compliance --into ./claims.json
3

Search stored knowledge

Search across all packs and stored findings. Find relevant claims without re-researching.

$ npx @grainulator/silo search "encryption at rest"
Real example

Pulling HIPAA constraints into a healthcare sprint

Your team is researching a patient data feature. Instead of starting from scratch, pull the compliance pack.

$ npx @grainulator/silo packs Pack Claims Types Tier ------------------------------------------------------- compliance 14 constraint documented migration 10 risk + constraint mixed architecture 12 all types mixed $ npx @grainulator/silo pull compliance --into ./claims.json Pulled 14 claims into ./claims.json Added: 6 HIPAA constraints, 4 SOC 2 constraints, 4 GDPR constraints All claims are type "constraint" with tier "documented" Sources: HHS.gov, AICPA TSC, GDPR Article references $ npx @grainulator/silo search "encryption at rest" Found 3 matches: compliance/hipaa-07 PHI must be encrypted at rest (AES-256 or equivalent) compliance/soc2-03 Data at rest must use encryption meeting CC6.1 criteria architecture/a09 Transparent encryption adds <2% latency for SSDs
Key features

Stop researching the same things

Knowledge you can reuse, version, and share across teams.

Knowledge Packs

Version-controlled bundles of claims. Compliance, migration risks, architecture patterns -- all pre-researched.

Clean Merge

Pull claims into any sprint. IDs are namespaced so they never conflict with your existing research.

Full-Text Search

Search across all stored knowledge. Find relevant claims without knowing which pack they are in.

Team Sharing

Store your best sprint findings as reusable packs. Next team gets a head start, not a blank slate.

Filesystem Storage

Everything is JSON files in ~/.silo. Inspect, back up, or version control with standard tools.

Zero Dependencies

Node built-ins only. No database, no network calls, no accounts. Works offline forever.
